Documentos HTML

Todo documento HTML debe iniciar con la declaración <!DOCTYPE html>.

El documento HTML inicia con el tag <html> y termina con </html>.

La parte visible de un documento HTML se encuentra en el tag body.

Elementos

<head>

El elemento head es un contenedor de metadatos que se inserta antes del body y la información no se muestra. Los metadatos típicamente definen title, character set, styles, links, scripts y otros metadatos.

Tags ejemplos:

  • <title>
  • <style>
  • <meta>
  • <link>
  • <script>
  • <base>

<title>

Este elemento define el título del documento y es requerido en todo documento HTML/XHTML.

El elemento define:

  • El título en la pestaña del buscador
  • Título en caso de que se agregue a favoritos
  • Muestra el título para la búsqueda de la página

Ejemplo:
<head>
<title>Page Title</title>
</head>

<meta>

Es usado para especificar qué character set es usado, descripción de la página, palabras clave, autor y otros metadatos.Define una descri[ción de la página web.

Ejemplo:
<meta charset="UTF-8">

<base>

Especifica la URL base y el target base para todos los URLs relativos.
<base href="https://www.w3schools.com/images/" target="_blank">



Headings

Los headings en HTML se definen desde <h1> hasta <h6>.

h1 define al heading más importante mientras que h6 define al menos importante.

Párrafos

Los párrafos en HTML se definen con la etiqueta <p>.

No es necesario utilizar el tag final pero se recomienda siempre utilizarlo.

Ejemplo:
<!DOCTYPE html>
<html>
<body>
<h1>My First Heading</h1>
<p>My first paragraph.</p>
</body>
</html>

Links

Los links en HTML se definen con la etiqueta a y el <link> de destino se inserta con el atributo herf.

Ejemplo:
<a href="https://www.w3schools.com/html/">Visit our HTML tutorial</a> Visita w3Schools

También se utiliza para linkear hojas de estilo externas.
<link rel="stylesheet" href="mystyle.css">

Un link local (en el mismo sitio) se especifica con una URL relativa (sin http://www....).

Colores de link

Por default, un link aparece de la siguiente manera (en todo buscador):
Link no visitado subrayado en azul.
Link visitado subrayado en morado.
Link activo subrayado en rojo.

El atributo target

El atributo target define dónde abrir el documento del link dentro del elemento herf.

El target puede tener los sigueintes valores:
_blank - Opens the linked document in a new window or tab
_self - Opens the linked document in the same window/tab as it was clicked (this is default)
_parent - Opens the linked document in the parent frame
_top - Opens the linked document in the full body of the window
framename - Opens the linked document in a named frame

Ejemplo: <a href="https://www.w3schools.com/" target="_blank">Visit W3Schools!</a>

Imágenes en HTML

En este caso, las imágenes se definen con el tag img seguido de sus atributos.

Ejemplos:
<!DOCTYPE html>
<html>
<body>
<h2>Spectacular Mountain</h2>
<img src="pic_mountain.jpg" alt="Mountain View" style="width:304px;height:228px;">
</body>
</html>

Todos los elementos de los que hemos hablado inician con una etiqueta y terminan de igual forma.

Existen elementos vacíos en HTML que se definen con el tag br. Este tag define el line break cuando no se quiere utilizar otro párrafo..

Los tags no distinguen entre mayúsculas o minúsculas, sin embargo es preferible utilizar minúsculas para los tags.

Atributos

Declaración del lenguaje en HTML

El lenguaje se declara dentro de la etiqueta html con el atributo lang.

Atributo title

El atributo title se añade dentro del elemento p. El atributo aparecerá como pequeña pestaña cuando el cursor se encuntre encima del párrafo.

Atributo herf

Los links en HTML se definen con la etiqueta a. La URL se inserta en el atributo herf que se encuentra dentro de la etiqueta.

Atributos de tamaño

Dentro de la etiqueta img se insertar como atributos el src (source) y el tamaño de la imagen (width and height).

El tamaño de la imagen está especificado en pixeles.

Atributo alt

Este atributo especifica una alternativa de texto a mostrar cuando una imagen no se puede mostrar.

El atributo alt de inserta de igual manera en la etiqueta img.

Al igual que los tags, se sugiere utilizar minúsculas para los atributos.

Todos los elementos en HTML pueden tener atributos.

Se recomienda utilizar comillas para los valores de los atributos.

Lista de atributos

alt: Alternativa de texto cuando una imagen no se puede mostrar.

disabled: Especifica que un elemento de entrada debe ser desactivado.

href: Especifica la URL(dirección electrónica) para un link o enlace.

id: Especifica un id único para un elemento.

src: Especifica la URL para una imagen.

style: Especifica un estilo CSS para un elemento.

title: Especifica información extra de un elemento.

Comentarios

Los etiqueto de comentarios se utiliza para insertar comentarios en en código HTML.

Ejemplo:
<!-- Write your comments here -->

Comentarios condicionales

Solo son ejecutados por Internet Explorer.

Crear un bookmark

Son utilizados para saltar a una parte específica de la página web.

Se debe crear un bookmark y después añadir un link.

Cuando se de click en el bookmark, la página se desliza a la ubicación del bookmark.

Ejemplo: h2 id="C4">Chapter 4 /h2
a href = "#C4"> Jump to Chapter 4 /a

Camino externo

Las página externas pueden ser referenciadas con un URL completo o con un camino relativo a la página actual.

<a href="https://www.w3schools.com/html/default.asp">HTML tutorial </a>
<a href="/html/default.asp">HTML tutorial </a>

Listas en HTML

Lista desordenada

Una lista ordenada empieza con el tag ul. Cada item de la lista empieza con li.

Lista ordenada

Una lista ordenada empieza con el tag ol. Cada item de la lista empieza con li.

Listas de descripción

Es una lista de terminos con una descripción por término. El tag dl define la lista de descripción. El tag dt define el término (nombre) y el tag dd describe cada término.

Listas horizontales

Se utlizan con CSS.

Bloques

Block-level elements

Un block-level element siempre empieza en una nueva linea y toma la anchura disponible.

Ejemplo:
<div style="background-color:black;color:white;padding:20px;">
<h2>London</h2>
<p>London is the capital city of England. It is the most populous city in the United Kingdom, with a metropolitan area of over 13 million inhabitants.</p>
</div>

Primer <div>

Este div contiene un style.

Ejemplos:

  • div
  • h1-h6
  • p
  • form

Elementos en linea

Un elemento en linea no empieza en una nueva linea y solo toma la anchura necesaria.

Ejemplo: <a href="https://www.w3schools.com/" target="_blank">Visit W3Schools!</a></p>

Ejemplos:

  • span
  • a
  • img

El elemento div

Es generalmente utilizado como contenedor de otros elementos HTML.

Este elemento no requiere atributos pero style y class son comunes. Se utiliza en conjunto con CSS.

El elemento span

Es usualmente utilizado como contenedor de algún texto.

El elemento no requiere atributos pero style y class son comunes.

IFrames

Un IFrame es usado para mostrar una página web en una página web. Se define con el tag iframe con el atributo src y la URL.

Un iframe utiliza los atributos height y width para especificar el tamaño del iframe.

Ejemplo:
<iframe src="demo_iframe.htm" name="iframe_a"></iframe>
<p><a href="https://www.w3schools.com" target="iframe_a">W3Schools.com</a></p>

Remover el borde

Un iframe tiene un borde alrededor por default.

Para remover el borde, se agrega un style y esa la propiedad CSS de border.

Objetivo para un link

El atributo target de un link debe referirse al atributo name de un iframe.

Camino de archivo

Un file path describe la localización de un archivo en un sitio web.

File paths son utilizados cuando se linkea a un archivo externo como:

  • Web pages
  • Images
  • Style sheets
  • JavaScripts

Absolute File Paths

Un file path absoluto es el URL completo a un archivo de internet.

Camino relativo a un archivo

Un relative file path apunta a un archivo relativo dentro de la página actual.

Establecer el Viewport

El viewport es la parte visible por el usuario. Varía con el dispositivo.

Ejemplo:
<meta name="viewport" content="width=device-width, initial-scale=1.0">

HTML Forms

Elemento <form>

Es un elemento que define una forma para capturar un datos que ingresa el usuario.

Los elementos form son diferentes tipos de input, como text fields, checkboxes, radio buttons, submit buttons y más.

Elemento <input>

Es el elemento más importante de un elemento form.
Ejemplos:

  • <input type="text">
  • <input type="radio">
  • <input type="submit">

Elemento <select>

Este elemento define un DDL (drop-down list).
Se utiiza el elemento option con un value. Para definir un elemento pre seleccionado, se añade el atributo selected a la opción.

Elemento <textarea>

Define un campo multi línea (text area). Utiliza los atributos rows(lineas) y cols(width).

Elemento <button>

Define un botón para hacer click.

Atributo Action

Define la acción a realizar cuando se ejecuta el form.

Atributo Method

Especifica el método HTTP (GET o POST) a utilizar.

¿Cuándo usar GET?

Es el método default al usar un submit. Se utiliza para información NO sensible.

¿Cuándo usar POST?

Al contrario del GET, se utiliza para manejar información sensible. No muestra la información.

Atributo Name

Cada campo input debe llevar un atributo nombre para ejecutarse. Si no se utliza el atributo, el imput no será enviado.

Agrupando Form Data con <fieldset>

Es utilizado para agrupar datos relacionados en un form.

El elemento legend define un encabezado para el fieldset.

Lista de atributos para form

  • accept-charset Specifies the charset used in the submitted form (default: the page charset).
  • action Specifies an address (url) where to submit the form (default: the submitting page).
  • autocomplete Specifies if the browser should autocomplete the form (default: on).
  • enctype Specifies the encoding of the submitted data (default: is url-encoded).
  • method Specifies the HTTP method used when submitting the form (default: GET).
  • name Specifies a name used to identify the form (for DOM usage: document.forms.name).
  • novalidate Specifies that the browser should not validate the form.
  • target Specifies the target of the address in the action attribute (default: _self).

Lista de atributos para inputs

  • value: Especifica el valor inicial para un input field.
  • readonly: Especifica solo se puede leer.
  • disabled: Especifica que el input field está desactivado.
  • size: Especifica el tamaño en caracteres del input field.
  • maxlength: Especifica el tamaño máximo de un input field.
  • autocomplete: Cuando está encendido, el browser autocompleta con valores antes introducidos por el usuario.
  • novalidate: Especifica que al enviar no sea validada la información.
  • autofocus: Especifica que el input field automáticamente se enfoca cuando carga la página.
  • form: Especifica un o que uno o más inputs pertenecen a un form.
  • formaction: Especifica la URL o file que se procesará en input control cuando el form se ejecute. Sobreescribe el atributo action.
  • formenctype: Especifica cómo se deben codificar los datos cuando se envía la información (solo forms con method=post).
  • formmethod: Define el method HTTP para enviar form-data a un action URL.
  • formnovalidate: Sobreescribe el novalidate del form.
  • formtarget: Especifica un name o keyword que indica dónde mostrar la respuesta recibida del submitting form.
  • list: Se refiere a un <datalist> que contiene opciones predefinidas para un input element.
  • multiple: Especifica que el usuario tiene permitido ingresar más de un valor en el input.
  • placeholder: Especifica un pista que describe el valor esperado en el input field.
  • required: Especifica que un inputfield debe ser llenado antes de enviar la información.
  • step: Especifica el intervalo numérico para un input.
